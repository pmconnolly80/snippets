This view is visible to logged-in users only: our code watches the $stateChangeStart event and if access to the next state (aka "route") is not authorized (based on the current user roles and on the access parameters defined in the states), navigation to the route is prevented with event.preventDefault();. This is a CLIENT-SIDE protection and as such is limited as users can still tamper with it on the client.

The following data is returned by the server to logged-in users only. This is more secure as the user needs to have an active session ON THE SERVER to obtain the data.

Try logging out in another tab and hit Refresh to see what happens.
